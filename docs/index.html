GodSpeed

Launch your own self-hosted AI workspace in one command.

The installer clones or updates the app, selects a tested Python runtime, repairs broken virtual environments, installs dependencies, runs setup, starts optional ChromaDB, opens the browser, and writes the Chrome assistant config.

Installers

One-go run commands

Use these for fresh installs, updates, and repair runs. Default native URL: http://127.0.0.1:7860.

macOS / Linux
curl -fsSL -H "Cache-Control: no-cache" https://raw.githubusercontent.com/pichimail/godspeed/main/scripts/install-godspeed.sh | bash
Windows PowerShell
$p="$env:TEMP\install-godspeed.ps1"; iwr -UseB -Headers @{"Cache-Control"="no-cache"} https://raw.githubusercontent.com/pichimail/godspeed/main/scripts/install-godspeed.ps1 -OutFile $p; powershell -ExecutionPolicy Bypass -File $p
Docker-only
git clone https://github.com/pichimail/godspeed.git
cd godspeed
cp .env.example .env
docker compose up -d --build

Self-repairing native install

Reuses or updates the existing install, rejects unsupported Python 3.14, and rebuilds the venv with Python 3.11–3.13.

Vector features handled

If Docker is installed and running, the installer starts bundled ChromaDB. If not, GodSpeed still launches with vector memory/RAG degraded.

Browser opens directly

The native installer starts Uvicorn in the background, writes logs, opens GodSpeed, and prepares Chrome assistant config.

Useful variants

Ports, LAN, and logs

Custom native port
GODSPEED_PORT=7870 bash <(curl -fsSL -H "Cache-Control: no-cache" https://raw.githubusercontent.com/pichimail/godspeed/main/scripts/install-godspeed.sh)
LAN / Tailscale bind
GODSPEED_HOST=0.0.0.0 bash <(curl -fsSL -H "Cache-Control: no-cache" https://raw.githubusercontent.com/pichimail/godspeed/main/scripts/install-godspeed.sh)
macOS/Linux logs: ~/.godspeed/app/logs/godspeed.log Windows logs: $HOME\.godspeed\app\logs Docker URL: http://localhost:7000
